法术标记检测系统增强计划

目标
- 将法术标记检测从“正则 + 连续重复”升级为“序列模式 + 多候选 + 通配 + 忽略空格”
- 提供可持久化的用户自定义规则，并保留可更新的内置规则
- 先稳定引擎与配置，再逐步完善图形化编辑器

一、范围与原则
- 内置规则与用户规则分离存储，运行时合并
- 明确匹配优先级与重叠处理
- 先实现最小可用编辑入口，后续再做可视化槽位编辑

二、数据模型
新增类型
- PatternSlot: { alternatives: string[] }
- SpellMarkingRule:
  - id, name, pattern: PatternSlot[]
  - matchMode: 'exact' | 'repeat' | 'contains'
  - minRepeat: number
  - ignoreEmpty: boolean
  - color, label?, enabled, isBuiltIn?

内置规则示例
- divide_chain
  - pattern: [{ alternatives: ['DIVIDE_*'] }]
  - matchMode: 'repeat'
  - minRepeat: 2
  - ignoreEmpty: false

设置存储
- AppSettings 增加 userMarkingRules?: SpellMarkingRule[]
- 内置规则不持久化，仅运行时合并

三、检测引擎
新函数
- matchSlot(slot, spellId): 支持精确匹配与前缀通配
- normalizeSpellId: 大小写统一
- detectPatterns(spells, deckCapacity, rules): 支持三种模式与 ignoreEmpty

匹配语义
- exact: 仅当模式长度与目标序列长度一致时命中
- contains: 在序列中滑窗匹配
- repeat: 以“pattern 作为整体”重复匹配

重叠处理
- 默认“先命中先占位”，禁止重叠
- 若有多条规则命中同一槽位，按规则排序决定优先级

四、UI 与交互
阶段 1（最小可用）
- Settings 中新增“法术标记规则”区域
- 支持：
  - 启用/禁用
  - 编辑名称、颜色、模式、忽略空格
  - 文本模式输入（例如：DIVIDE_* | DIVIDE_2 + LIGHT_BULLET）

阶段 2（图形化编辑）
- 轻量槽位编辑器（不复用 WandEditor）
- 每个槽位显示多个候选图标
- 点击槽位弹出迷你 Picker 添加候选
- 右键/点击候选图标移除
- 支持输入通配符候选

五、集成点
- WandEditor 读取 settings 中“合并后的规则”
- 仅对 enabled 规则进行检测

六、国际化
- 新增标记规则相关文案（zh/en）

七、测试与验证
- 单元测试：matchSlot / detectPatterns
- 覆盖：
  - 通配符
  - ignoreEmpty
  - repeat + 多槽位
  - 重叠优先级
- 手动验证：UI 规则编辑与高亮结果
